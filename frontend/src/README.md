# 前端架构说明

## 目录职责

- `App.vue`：应用壳层，只负责登录态、页面选择和全局错误承接。
- `pages/`：页面级业务模块，一个页面一个文件，承载页面自己的查询、筛选、提交和 loading 状态。
- `components/`：跨页面复用组件，只做展示和轻量交互，不直接持有业务 API 编排。
- `constants/`：业务常量和纯函数，例如生成状态、尺寸、比例、轮询间隔。
- `utils/`：浏览器侧通用工具，例如下载、参考图文件读取。
- `lib/api.js`：后端接口封装和统一响应拆包。

## 扩展约定

- 新增独立页面时放到 `pages/`，由 `App.vue` 注册入口。
- 新增跨页面控件时放到 `components/`，通过 props / emits 通信。
- 页面内私有状态优先留在页面文件中；多个页面复用后再抽 composable 或 store。
- 不在组件里展示接口路径、模型字段等技术细节给普通用户。
- PC 端优先使用高信息密度表格，移动端通过 CSS 转为卡片式列表。
